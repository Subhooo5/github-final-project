# Contributing

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## Bug Reports

Open an issue and include:

- What you ran, and what you expected to happen
- What actually happened, including any error output
- Your operating system and Bash version (`bash --version`)

## Bug Fixes

1. Fork the repository.
2. Create a branch: `git checkout -b fix/short-description`
3. Make the change and test the script locally.
4. Commit with a clear message describing the fix.
5. Push the branch and open a pull request.

## Documentation Improvements

Corrections to the README, clarifications to usage instructions, and typo fixes are all valuable. Submit them as pull requests the same way as code changes.

## Enhancements and Ideas

Open an issue describing the proposed change before writing code, so the approach can be discussed first.

## Code Style

- Start scripts with `#!/bin/bash`
- Use lowercase variable names with underscores
- Quote variable expansions
- Keep the script readable over clever

## Code of Conduct

Participation in this project is governed by the [Code of Conduct](CODE_OF_CONDUCT.md).
